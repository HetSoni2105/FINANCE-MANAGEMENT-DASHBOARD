# Finance Management Dashboard - Development Walkthrough

## Project Rules

### Rule 1

Do not optimize until the project is fully functional.

### Rule 2

When debugging:

* Investigate causes first.
* Trace data flow.
* Understand why the issue occurs.
* Avoid immediately requesting solutions.

### Rule 3

After every completed step:

* Verify requirements.
* Verify data flow.
* Verify future expansion compatibility.
* Verify component responsibility.

### Rule 4

Build only what is required for the current phase.
